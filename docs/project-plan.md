# План проекта Thermal Panels

## 1. Краткое резюме требований

- Проект: fullstack-сайт для продажи декоративных термопанелей для облицовки и утепления фасада.
- Основная цель: привести пользователя к сценарию "Рассчитай стоимость своего дома" и к прямому контакту по телефону или через мессенджеры.
- Формат сайта: светлый, современный, адаптивный лендинг с каталогом, калькулятором и административной панелью.
- Онлайн-оплата не предусмотрена.
- Критичные функции: каталог товаров, калькулятор стоимости, управление контентом и товарами через админ-панель.
- Обязательные UX-условия: фиксированный header, заметные контактные кнопки, повторение ключевого CTA в нескольких местах, простой мобильный сценарий.
- Контент должен быть убедительным, но без юридически рискованных абсолютных обещаний по экономии энергии, шумоизоляции и характеристикам, если они не подтверждены.

## 6. Минимальная fullstack-архитектура

### Технологический минимум

- Frontend: React на существующем Vite.
- Backend: Node.js + Express.
- Database: PostgreSQL.
- API: REST.
- Auth: один администратор без ролевой модели.
- Media: локальное файловое хранилище на сервере.

### Почему это минимально и достаточно

- текущий frontend уже основан на Vite + React, поэтому стек сохраняется;
- Express подходит для простого REST API без лишней архитектурной нагрузки;
- PostgreSQL покрывает каталог, контент, контакты, медиа-метаданные и администратора;
- локальное хранение файлов подходит для первой версии и не требует облачной инфраструктуры.

## 8. Минимальные интерфейсы API

### Публичные endpoints

- `GET /api/products` — список товаров каталога.
- `GET /api/products/:slug` — детальная карточка товара.
- `POST /api/calculator/estimate` — предварительный расчёт стоимости.
- `GET /api/site-content` — тексты, контакты, блоки лендинга, объекты.

### Admin endpoints

- `POST /api/admin/auth/login` — вход администратора.
- `GET /api/admin/products` — список товаров для админки.
- `POST /api/admin/products` — создание товара.
- `PUT /api/admin/products/:id` — редактирование товара.
- `DELETE /api/admin/products/:id` — удаление товара.
- `POST /api/admin/products/:id/images` — загрузка фото товара.
- `POST /api/admin/uploads/images/:scope` — загрузка изображения в локальное файловое хранилище.
- `GET /api/admin/showcase-objects` — список объектов.
- `POST /api/admin/showcase-objects` — создание объекта.
- `PUT /api/admin/showcase-objects/:id` — редактирование объекта.
- `DELETE /api/admin/showcase-objects/:id` — удаление объекта.
- `PUT /api/admin/site-content/:blockKey` — обновление текстов блока.
- `PUT /api/admin/contacts` — обновление контактов и ссылок.

## 9. Минимальная логика расчёта и скидок

### Скидки

- Если у товара заданы старая и новая цена, система рассчитывает скидку автоматически.
- Формула: `((oldPrice - currentPrice) / oldPrice) * 100`.
- Результат округляется до целого числа.
- В каталоге и карточке показываются зачёркнутая старая цена, новая цена и бейдж скидки.

### Калькулятор

- Пользователь выбирает товар.
- Пользователь вводит площадь фасада в м².
- Пользователь выбирает вариант монтажа: самостоятельно или с расчётом монтажа.
- Система учитывает полезную площадь панели выбранного товара.
- Система добавляет запас.
- Система рассчитывает ориентировочное количество панелей и итоговую стоимость.
- Результат маркируется как предварительный.
